Project Overview

The Critical Code Reviewer (CCR) Framework is a responsible AI initiative designed to help educational technology developers identify potential fairness, accessibility, inclusion, pedagogy, transparency, and privacy concerns during software development and code review.

Traditional code-review tools often focus primarily on functionality, performance, security, and maintainability. CCR extends this process by evaluating code through a human-centered and ethical lens, helping developers consider how software decisions may affect diverse learner populations.

The project was developed within the Arizona State University Learning Engineering Institute to support the creation of more equitable, inclusive, transparent, and learner-centered educational technologies.

Problem Statement

Educational technologies increasingly influence learning experiences, recommendations, assessments, progression decisions, and access to educational opportunities. However, many software-development workflows do not systematically evaluate the ethical and social implications of code before deployment.

CCR addresses this gap by translating responsible AI and inclusive-design principles into practical dimensions, criteria, definitions, and reviewable indicators that developers can apply during code review.

Project Goals

  • Develop an evidence-informed framework for ethical and responsible code review.
  • Translate responsible AI principles into practical guidance for software developers.
  • Support inclusive and equitable educational technology development.
  • Help developers recognize fairness, accessibility, transparency, privacy, and learner-impact risks in code.
  • Support the development of an AI-assisted code-review tool for educational technology contexts.
  • Validate and refine the framework using feedback from educational technology stakeholders and software developers.

Framework Dimensions

Fairness Evaluation

Examines whether training and evaluation data adequately represent the learner populations and educational settings the system is intended to serve. It also considers whether group-level fairness targets and evaluation procedures are defined.

Pedagogy

Evaluates whether adaptive educational systems support meaningful learner progression, transparent learning requirements, appropriate early-warning decisions, and pedagogically grounded recommendations.

Decision Fairness

Assesses how protected or proxy attributes are used in decision logic and whether fairness protections are clearly defined, validated, enforced, and separated from inappropriate decision inputs.

Inclusion

Identifies culturally narrow assumptions, exclusionary language, inaccessible user flows, and design decisions that may disadvantage learners with different identities, abilities, devices, languages, locations, or levels of connectivity.

Transparency

Evaluates whether AI-driven outputs are understandable, explainable, appropriately qualified, and supported by human review, contestability, and override mechanisms.

Privacy

Assesses whether learner data are collected, processed, stored, and shared with appropriate consent checks and safeguards.

Example Evaluation Criteria

  • Unrepresentative Dataset Coverage
  • Undefined Fairness Evaluation Attributes
  • Missing Group-Wise Fairness Targets
  • Unvalidated Early-Warning Thresholds
  • Persistent Negative State
  • Hidden Learning Path Requirements
  • Protected and Proxy-Based Differential Treatment
  • Unvalidated Fairness Mitigations
  • Culturally Narrow Representations and Validation
  • Exclusionary Language in Code and UI
  • Excluding Access Assumptions
  • Opaque AI Decision Outputs
  • Causal Overclaiming in Analytics
  • Automation Without Review Safeguards
  • Missing Consent Checks for Learner Data

Framework Validation Study — Summer 2026–Present

Following the development of the initial Critical Code Reviewer Framework, I am currently contributing to a user-centered validation study designed to evaluate and refine the framework through feedback from educational technology stakeholders and software developers.

The study investigates whether the framework adequately addresses responsible AI concerns, including fairness, accessibility, inclusion, transparency, privacy, accountability, and trust.

It also evaluates the framework's clarity, usefulness, completeness, conceptual organization, and practicality during software code review.

Research Design

The validation study uses a sequential, qualitative-dominant mixed-methods design consisting of two phases.

Phase 1: Educational Technology Stakeholders

  • Semi-structured interviews
  • Demographic questionnaire
  • Identification of stakeholder concerns
  • Reflexive thematic analysis
  • Refinement of the CCR Framework into Version 2.0

Phase 2: Software Developer Evaluation

  • Hybrid card sorting using FigJam
  • Think-aloud usability testing
  • System Usability Scale
  • Framework-specific evaluation survey
  • Development of Version 3.0 of the CCR Framework
